dsu on tree 第一轮复习

持续更新 | 发表于 2019-08-13 更新于 2019-08-29 分类于 Notes 阅读次数： Valine：

简介

dsu on tree，即树上启发式合并，利用重链剖分的性质保证复杂度。

流程

首先考虑一个离线静态树上的子树查询问题。

最暴力的思路是这样的：

  • 枚举树上的点 \(i\)
    • 暴力 DFS 统计整棵子树的答案
    • 回答这个点的所有查询
    • 清空答案

这样做显然是 \(O(n^2)\) 的。

注意到每次清空统计结果会降低效率。所以我们考虑对于每个点，不清除它的某个儿子的答案：

  • 从根节点开始 DFS，当前 DFS 到了 \(x\) 号点
    • 找一个儿子 \(y\)
    • DFS 除 \(y\) 以外的其它节点，并清除它们的答案
    • DFS \(y\) 号点
    • 继承 \(y\) 的答案
    • 暴力统计除 \(y\) 以外其它子树的答案
    • 回答 \(x\) 的所有查询

令 \(y\) 的子树大小为 \(m\)，\(x\) 的子树大小为 \(n\)。那么有：\(T(n)=T(m)+T(n-m)+O(n-m)\)。

最坏情况下，\(m=n-m=\frac n2\)。可得 \(T(n)=2T(\frac n2)+O(n)=O(n\log n)\)。

显然，\(y\) 应该取 \(x\) 的重儿子。

例题

CF570D Tree Requests

题意

给一棵树，每个点上有个字母。你要支持查询一个数对 \((x,y)\)，即查询 \(x\) 子树所有在原树深度为 \(y\) 的点上的字母是否可以重排成一个回文串。

\(1\le n,m\le5\times10^5\)。

解法

令 \(c_i\) 为 当前点 深度为 \(i\) 时，每个字母出现次数的奇偶性状态压缩的值。

那么当 \(c_i=0\) 或 \(2^a\) 时，当前点 深度为 \(i\) 的节点上的字母可以重排成一个回文串。

考虑 dsu on tree。每次继承重儿子上的答案，暴力统计时算出 \(c\) 的值。复杂度 \(O(n\log n)\)。

CF741D [已编辑]

题意

名字太长了，删了。

给一棵有根树，每条边上有一个 a~v 的字母。如果一条路径上所有边的字母重排后可以组成一个回文串，那么这条路径被称为「迷之路径」。求每个子树下最长的「迷之路径」。

\(1\le n\le5\times10^5\)。

解法

考虑 dsu on tree。

用类似上一题的思路，记录 \(path_x\) 为 \(x\) 到根路径上所有字母出现次数奇偶性状压后的值。

那么如果 \(path_x\oplus path_y\) 等于 \(2^a\) 或 \(0\)，可以推出 \(x\leftrightarrow y\) 是一条「迷之路径」。

所以可以继承所有子树的答案，然后只需要统计经过根的路径。

我们开一个 \(dis_S\)，代表所有满足 \(path_x=S\) 中最深的 \(x\)。

然后用 dsu on tree 的思路，继承重儿子的 \(dis\) 数组，然后枚举所有的轻儿子，计算子树里所有点的答案 之后 将这个子树加入 \(dis\) 数组。

注意，如果边计算答案边统计 \(dis\) 数组可能会将同一个子树的两个节点统计入答案。这些答案已经在递归子树时统计过了，而且如果重复统计这些点会让复杂度乘上 LCA 的查询时间。
